# 需求分析文档 — T1-1 命令行待办工具 (Simple Todo CLI)

## 1. 项目概述

开发一个简单的命令行待办事项工具 `todo.py`，通过命令行参数支持任务的添加、列出和删除操作。数据持久化到文本文件 `todos.txt`。

## 2. 功能需求

### FR-1: 添加任务 (`add`)

- **命令格式**: `python todo.py add "任务内容"`
- **行为**: 将任务内容追加到 `todos.txt` 文件末尾，作为新的一行
- **特殊要求**:
  - 任务内容可能包含特殊字符（如 `!@#$%^&*()`），必须原样保存和显示，不得转义或丢弃
  - 添加成功后进程退出码为 0
- **边界条件**:
  - `add` 缺少任务内容参数时（如 `python todo.py add`），应静默忽略或退出码为 0，不应崩溃

### FR-2: 列出任务 (`list`)

- **命令格式**: `python todo.py list`
- **行为**: 逐行打印所有任务，格式为 `ID: 任务内容`，其中 ID 为从 1 开始的行号
- **边界条件**:
  - 当 `todos.txt` 不存在时，视为空列表
  - 当没有任务时，**不应输出任何内容**（stdout 为空字符串），进程退出码为 0

### FR-3: 删除任务 (`remove`)

- **命令格式**: `python todo.py remove <ID>`
- **行为**: 根据行号（1-based）删除对应的任务行
- **边界条件**:
  - 如果 ID 不存在（超出范围或非正整数），**不应崩溃**，静默忽略，进程退出码为 0
  - 删除后，剩余任务的 ID 自动重新编号（ID 始终为 1-based 行号）

## 3. 数据模型需求

### DM-1: 存储格式

- **文件**: `todos.txt`，位于**进程的工作目录**。由于测试从 `workspace/` 目录运行（`subprocess.run(["python3", "starter/todo.py", ...])`），而代码中 `DB_FILE = "todos.txt"` 使用相对路径，因此 `todos.txt` 实际创建在 `workspace/` 目录下（注意：不是 `starter/` 目录）
- **格式**: 纯文本，每行一个任务内容（无前缀、无额外空行）
- **ID 映射**: 任务 ID = 过滤空行后的任务在列表中的索引（从 1 开始）。注意：`load_todos()` 会过滤掉空行（`if line.strip()`），因此 `todos.txt` 中的空行不占用 ID

### DM-2: 文件状态

- `todos.txt` 不存在 → 等同于空任务列表
- `todos.txt` 为空文件 → 等同于空任务列表

## 4. 接口行为描述

| 命令 | 参数 | 正常行为 | 异常行为 |
|------|------|----------|----------|
| `add` | 任务内容字符串 | 追加到文件，退出码 0 | 缺少内容参数时静默忽略，不崩溃 |
| `list` | 无 | 打印 `ID: 内容`，退出码 0 | 文件不存在时输出为空 |
| `remove` | ID（整数） | 删除对应行，退出码 0 | ID 超出范围时静默忽略 |

## 5. 非功能需求

- **运行环境**: Python 3
- **依赖**: 无第三方依赖，仅使用 Python 标准库
- **退出码**: 所有命令在正常和异常情况下均应返回退出码 0

- `tests/test_todo.py` 已提供完整测试用例，不可修改
- 测试通过 `subprocess.run(["python3", "starter/todo.py", ...])` 运行，工作目录为 `workspace/`

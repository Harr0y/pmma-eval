# 需求规格说明 — T1-1 命令行待办工具 (Simple Todo CLI)

## 1. 概述

开发一个命令行待办事项工具 `todo.py`，支持添加、列出、删除任务，使用纯文本文件持久化存储。

## 2. 功能需求

### FR-1：添加任务 (`add`)

| 属性 | 说明 |
|------|------|
| 命令格式 | `python todo.py add "任务内容"` |
| 行为 | 将任务内容追加到存储文件末尾 |
| ID 分配 | 自动分配，基于行号（1-based） |
| 特殊字符 | 必须正确保存和显示包含 `!@#$%^&*()` 等特殊字符的任务内容 |
| 返回码 | 成功时返回 0 |
| 错误处理 | 无需处理（命令行参数由 shell 传递） |

### FR-2：列出任务 (`list`)

| 属性 | 说明 |
|------|------|
| 命令格式 | `python todo.py list` |
| 输出格式 | 每行一个任务，格式为 `ID: 内容`（ID 为 1-based 行号） |
| 空列表行为 | 没有任务时**不应输出任何内容**（空输出，stdout 为空字符串） |
| 返回码 | 成功时返回 0 |

### FR-3：删除任务 (`remove`)

| 属性 | 说明 |
|------|------|
| 命令格式 | `python todo.py remove <ID>` |
| 行为 | 删除指定 ID 的任务，后续任务 ID 自动重新编号 |
| 不存在的 ID | 如果 ID 不存在（超出范围），**不应崩溃**，静默忽略，返回码 0 |
| 返回码 | 成功时返回 0（包括 ID 不存在的情况） |

## 3. 技术需求

### TR-1：数据存储

| 属性 | 说明 |
|------|------|
| 存储方式 | 纯文本文件 `todos.txt` |
| 存储位置 | 当前工作目录 |
| 文件格式 | 每行一个任务（任务内容本身，不含 ID） |
| ID 映射 | ID = 行号（从 1 开始） |

### TR-2：程序入口

| 属性 | 说明 |
|------|------|
| 文件位置 | `starter/todo.py` |
| 执行方式 | `python todo.py <command> [args]` |
| 参数解析 | 使用 `sys.argv` 解析命令行参数 |

## 4. 验收标准

| 编号 | 验收项 | 验证方法 |
|------|--------|----------|
| AC-1 | 添加任务后可通过 list 查看 | `add` + `list`，验证 stdout 包含正确内容 |
| AC-2 | 删除任务后列表正确更新 | `add` + `remove` + `list`，验证删除项消失且 ID 重新编号 |
| AC-3 | 空列表时 list 无输出 | 无任务时 `list`，验证 stdout 为空 |
| AC-4 | 删除不存在的 ID 不崩溃 | `remove 99`，验证返回码为 0 |
| AC-5 | 特殊字符正确保存和显示 | `add "Special !@#$%^&*()"` + `list`，验证输出包含原始特殊字符 |
| AC-6 | 所有测试通过 | `python -m pytest tests/test_todo.py -v` 全部 PASS |

## 5. 非功能需求

| 编号 | 需求 |
|------|------|
| NFR-1 | 代码放置于 `starter/todo.py`，不得创建新的 Python 文件 |
| NFR-2 | 不引入外部依赖，仅使用 Python 标准库 |
| NFR-3 | 现有 `load_todos()` 和 `save_todos()` 函数应复用 |

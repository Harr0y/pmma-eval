## 2. 实现的功能列表

### 标签路由（routes_tag.py）— 4 个端点

| 端点 | 功能 | 状态码 |
|------|------|--------|
| POST /tags | 创建标签（name 必填唯一） | 201 / 400 / 409 |
| GET /tags | 获取所有标签列表 | 200 |
| PUT /tags/\<id\> | 编辑标签名称 | 200 / 404 / 400 / 409 |
| DELETE /tags/\<id\> | 删除标签（级联解除关联） | 200 / 404 |

### 文章路由（routes_article.py）— 6 个端点

| 端点 | 功能 | 状态码 |
|------|------|--------|
| POST /articles | 创建文章（title/body 必填） | 201 / 400 |
| GET /articles | 列出文章（支持 ?tag= 和 ?tag_id= 筛选） | 200 |
| GET /articles/\<id\> | 获取单篇文章 | 200 / 404 |
| POST /articles/\<id\>/tags | 为文章绑定标签（幂等） | 200 / 404 / 400 |
| GET /articles/\<id\>/tags | 获取文章的所有标签 | 200 / 404 |
| DELETE /articles/\<id\>/tags/\<tag_id\> | 解除关联（幂等） | 200 / 404 |

## 6. 已知问题与改进建议

### 无已知阻塞问题

所有 18 个测试用例全部通过，3 条验收标准全部满足。

### 改进建议（非阻塞）

1. **分页支持**：GET /articles 和 GET /tags 目前返回全部数据，大量数据时可考虑分页
2. **批量操作**：POST /articles 可扩展为支持批量创建
3. **标签统计**：可增加 GET /tags 返回每个标签下的文章数量
4. **持久化会话**：当前为内存 SQLite，可考虑配置外部数据库

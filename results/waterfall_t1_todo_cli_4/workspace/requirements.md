# 需求分析文档

## 1. 功能需求

### FR-1: 添加任务 (`add`)

- **命令格式**: `python todo.py add "任务内容"`
- **行为**: 将任务内容追加到存储文件末尾
- **特殊字符处理**: 任务内容可能包含 `!@#$%^&*()` 等特殊字符，必须原样保存和显示，不得丢失或转义错误
- **成功返回**: 进程退出码 0

### FR-2: 列出任务 (`list`)

- **命令格式**: `python todo.py list`
- **行为**: 打印所有任务，格式为 `{ID}: {内容}`，ID 为 1-based 行号
- **空列表行为**: 当没有任何任务时，不应输出任何内容（空输出，stdout 为空字符串）
- **成功返回**: 进程退出码 0

### FR-3: 删除任务 (`remove`)

- **命令格式**: `python todo.py remove <ID>`
- **行为**: 根据行号删除对应任务，后续任务 ID 自动前移
- **越界处理**: 如果 ID 不存在（超出范围或小于 1），不应崩溃，静默忽略即可，退出码仍为 0
- **成功返回**: 进程退出码 0

## 2. 数据模型需求

### DM-1: 存储格式

- **存储文件**: `todos.txt`（与 todo.py 同级目录）
- **格式**: 每行一个任务，行尾使用 `\n` 换行
- **ID 规则**: 任务 ID = 行号（从 1 开始）
- **空文件**: 删除所有任务后，文件应被清空（0 字节）

### DM-2: 文件不存在处理

- 如果 `todos.txt` 不存在，视为空列表，不报错

## 3. 接口行为描述

| 命令 | 参数 | 正常行为 | 异常行为 |
|------|------|----------|----------|
| `add` | `"内容"` | 追加任务到文件末尾 | 无需特殊处理 |
| `list` | 无 | 输出 `ID: 内容` 每行一个 | 空列表时无输出 |
| `remove` | `<ID>` | 删除对应行 | ID 越界时静默忽略 |

## 4. 验收标准

| 编号 | 验收项 | 通过条件 |
|------|--------|----------|
| AC-1 | 基本流程 | add → list → remove → list，数据正确 |
| AC-2 | 空列表 | `list` 命令在无任务时 stdout 为空 |
| AC-3 | 删除越界 | `remove 99` 不崩溃，退出码 0 |
| AC-4 | 特殊字符 | 包含 `!@#$%^&*()` 的任务能正确保存和显示 |
| AC-5 | ID 重排 | 删除中间任务后，后续任务 ID 自动前移 |
| AC-6 | 测试通过 | `tests/test_todo.py` 中所有测试用例通过 |

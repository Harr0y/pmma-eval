# 方案设计文档

## 1. 数据存储设计

### 1.1 存储文件
- **文件名**：`todos.txt`（与 `todo.py` 同目录，即项目根目录）
- **格式**：每行一个任务，行尾 `\n`
- **编码**：UTF-8（默认）

### 1.2 文件不存在时的行为
- `load_todos()` 已处理：文件不存在时返回空列表 `[]`
- **等效语义**：文件不存在 ≈ 空任务列表，无需额外处理

### 1.3 数据完整性
- `load_todos()` 使用 `line.strip()` 去除首尾空白，并过滤空行
- `save_todos()` 每行写入后追加 `\n`
- 特殊字符不做任何转义/编码处理，原样读写

## 2. 命令行接口设计

### 2.1 命令路由

```
python todo.py <command> [args]
```

| 命令 | 参数 | 行为 |
|------|------|------|
| `add` | `<任务内容>` (sys.argv[2]) | 追加任务到列表 |
| `list` | 无 | 打印所有任务 |
| `remove` | `<ID>` (sys.argv[2]) | 按 ID 删除任务 |

### 2.2 命令实现逻辑

#### `add` 命令
```
1. 检查 sys.argv 长度 ≥ 3，若不足则 print usage 并 return
2. 任务内容 = sys.argv[2]
3. 调用 load_todos() 加载现有列表
4. 将任务内容 append 到列表
5. 调用 save_todos() 保存
6. 无输出（stdout 为空）
```

**注意**：`sys.argv[2]` 是 shell 传递的单个参数，引号内的特殊字符由 shell 处理，Python 直接接收。

#### `list` 命令
```
1. 调用 load_todos() 加载列表
2. 遍历列表，对每个任务打印 "{index}: {content}"
   - index 为 1-based（enumerate(todos, start=1)）
3. 无任务时循环不执行，stdout 自然为空
```

#### `remove` 命令
```
1. 将 sys.argv[2] 转换为整数 id
2. 调用 load_todos() 加载列表
3. 边界检查：if 1 <= id <= len(todos)
   - 满足条件：删除 todos[id-1]
   - 不满足条件：静默忽略，不做任何操作
4. 调用 save_todos() 保存（无论是否删除）
5. 无输出（stdout 为空）
```

**异常处理**：
- `int()` 转换失败（非数字输入）→ 用 try/except 包裹，静默忽略
- ID 超出范围 → 边界检查，静默忽略

## 5. 设计决策

| 决策点 | 选择 | 理由 |
|--------|------|------|
| 特殊字符处理 | 原样读写，不做转义 | README 明确要求"正确保存和显示"，纯文本存储天然支持 |
| 错误处理策略 | try/except 静默吞掉异常 | 需求要求"不应崩溃"且"静默忽略" |
| remove 无效 ID 时不写文件 | 仍然调用 save_todos() | 保持一致性，避免分支遗漏，且对空列表调用 save 也是安全的 |
| ID 类型 | 整数 | 需求和测试均使用整数 ID |
